# 대규모 언어모델 정리

## 현재 우리가 쓰는 언어모델
- 전부 __autoregressive__ 모델임
확률의 연쇄법칙을 그대로 적용한 것. 
다음단어를 계속 예측하는 것이 모델의 핵심

## 언어모델의 작동방식
1. 각 단어들을 토큰화해서 각각 ID를 부여
2. ID들을 신경망에 통과시킴
3. 다음 토큰에 대한 확률 분포를 출력
4. 거기서 샘플링해서 새 토큰 선택
5. 다시 역토큰화해서 단어로 변환

## 토크나이저
1. 큰 컨텍스트 코퍼스로 시작
2. 처음엔 모든 문자에 각각 토큰을 부여
3. 가장 자주 등장하는 토큰 쌍을 찾아서 합침
4. 반복
- 공백 이나 구두점 처리는 사전 토크나이저가 먼저 처리
### 토크나이저가 왜 중요한가 ? 
- 파이선에서 들여쓰기에 쓰는 공백 네개가 이전에는 이상하게 처리되었는데 이기능이 코드를 방해함. 327같은 숫자가 한의 토큰으로 처리되면 모델은 추리처럼 각 자릿수를 분리해서 생각할 수 없어짐

## Perplexity (언어모델의 대표적 평가 지표)
1. 검증 손실을 조금 더해석하기 쉽게 변한한 것
  - 평균 토큰 당 손실을 구하고 거기다 지수를 취함
    - 왜 지수를 취하는가? 로그는 사람이 직관적 이해 어려움
 2. 모델이 다음 단어를 예측할 때 몇개의 단어 사이에서 망설이는지를 나타냄 
    - Perplexity = 1 이면 완벽한 모델이다. 항상 정확한 단어를 예측할 수있다
    - Perplexity = 어휘 크기라면 완전 무작위 상태, 아무것도 모르는 상태

    ### 요즘 Perplexity를 잘 사용하지 않는 이유
    1. 토크나이저에 따라 값이 달라짐
    2. 어휘 토큰의 개수에따라 상한 자체가 달라져 단순비교가 불가능
    3. 현재는 MMLU같은 벤치마크를 개발단계에서 사용

## MMLU 벤치마크
- 가장 유명한 벤치마크 (대학의학, 물리학, 천문학, 다양한 분야의 4지선다 문제로 구성)
    ### 평가방법 
    1. 네가지 선택지 각각이 나올 확률을 보고 높은것을 선택
    2. 모델에게 A, B, C, D 중 하나를 고르라고 하고 다음 토큰이 뭔지 봄

    - 평가의 함정이 존재함
 같은 MMLU도 평가 방법에 따라 결과가 크게달라짐

    ### 벤치마크 오염문제
    - 모델이벤치마크 테스트셋으로 학습했을 때 우리는 무엇으로 학습했는지 알 수 없다. 
    - 탐지방법: 대부분의 데이터셋은 무작위 순서가 아님 모델이 순서대로 생성할 때 역순으로 생성할 때보다 학률이 높으면 그 데이터가 학습셋에 있었을 가능성이 높다.

## 데이터
- 대규모 언어모델이 인터넷에서 어떻게 학습하는가 ?
1. __1단계. 인터넷 전체다운로드__ 
    - Common Crawl 같은 웹크롤러 사용
    - 약 2500억개 페이지, 1페타바이트 데이터 6기가 바이트 짜리 압축 파일로 배포
2. __2단계. HTML에서 텍스트 추출__
    - HTML 태그 제거,반복요소 제거 수식 추출이 어려움
3. __3단계. 불필요 콘텐츠 필터링__
4. __4단계. 중복제거__
5. __5단계. 휴리스틱 필터링__
    - 저품질 문서 제거
    - 토큰 분포가 일반적인 것과 너무 다른 문서
    - 단어 길이가 너무 극단적인 문서
    - 단어가 세 개밖에 없는 문서
    - 단어가 1,000만 개인 문서
6. __6단계. 모델 기반 필터링__
    - 위키피디아에서 참조하는 링크들을 모음
    - 위키피디아를 기준으로 분류기를 학습시켜 품질 좋은 문서를 학습
7. __7단계. 도메인 분류 및 가중치 조정__
    - 데이터를 *엔터테이먼트*, *책*, *코드* 등으로 분류하고 가중치를 조정
    - (코드로 더 많이 학습시키면 추론능력이 올라간다고 알려져있다.)
8. __8단계 고품질 데이터로 파인튜닝__
    - 학습 마지막 단계에서 학습률을 낮추고 위키피디아 같은 고품질 데이터로만 학습을 진행함, 일종의 오버피팅으로 의도적으로 진행
    ### 왜 오버피팅을 하는가 ? 
        크게 3가지의 이유가 존재한다고 함
        1. Sanity Check
            1. 데이터가 적은데도 오버피팅이 불가능하다면, 모델 구현 자체가 논리적으로 틀렸거나 데이터가 모델에 제대로 전달되지 못하다는 증거가 될 수있음 
        2. Specialize Tasks
            - 새로운 데이터에 대한 대응력보다 기존 데이터의 완벽한 암기력에 대한 성능 지표확인하기 위해 
        3. Double Descent 
            - 최신 인공지능에서 오버피팅의 임계점을 넘어 모데을 극단적으로 복잡하고 파라미터의 수가 늘어난다면 초반에 성능이 나빠지다가 파라미터가 데이터 수보다 압도적으로 많아지는 지점을 넘어서면 모델이 더 똑똑해진다는 현상이 관찰됨 (모델은 클수록 좋다) 
    ### 데이터 규모 변화 
     - 초기 학술 모델 : 약 1500억 토큰(800GB)
     - 현재 최고 모델(SOTA) : 약 15조 토 (55TB~60TB)
## 스케일링법칙 
- 데이터가 많을수록, 모델이 클수록 예측이 가능한 방식으로 좋아짐 
- 모델을 크게 만들수록 성능이 계속 좋아지고 오버피팅이 없어진다 
    ### 스케일링 법칙의 형태
    - 컴퓨팅-손실, 데이터-손실, 파라미터-손실을 각각 로그-로그 스케일로 그리면 선형 관계가 그려진다. 즉 로그-로그 스케일에서직선이다. 컴퓨팅을 10배 느릴면 손실이 얼마나 줄어드는지 예측 할 수 있다는 뜻이된다.

        #### 이 포인트가 중요한 이유
        - 2~3년 후 모델이 얼마나 좋아질 지 예측이 가능하다. 컴퓨팅이 계속 증가할 거라는 걸 알면, 성능도 예측 가능하게 좋아질 것.
