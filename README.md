# 현재 우리가 쓰는 언어모델
- 전부 __autoregressive__ 모델임
확률의 연쇄법칙을 그대로 적용한 것. 
다음단어를 계속 예측하는 것이 모델의 핵심

## 언어모델의 작동방식
1. 각 단어들을 토큰화해서 각각 ID를 부여
2. ID들을 신경망에 통과시킴
3. 다음 토큰에 대한 확률 분포를 출력
4. 거기서 샘플링해서 새 토큰 선택
5. 다시 역토큰화해서 단어로 변환

## 토크나이저
1. 큰 컨텍스트 코퍼스로 시작
2. 처음엔 모든 문자에 각각 토큰을 부여
3. 가장 자주 등장하는 토큰 쌍을 찾아서 합침
4. 반복
- 공백 이나 구두점 처리는 사전 토크나이저가 먼저 처리
### 토크나이저가 왜 중요한가 ? 
- 파이선에서 들여쓰기에 쓰는 공백 네개가 이전에는 이상하게 처리되었는데 이기능이 코드를 방해함. 327같은 숫자가 한의 토큰으로 처리되면 모델은 추리처럼 각 자릿수를 분리해서 생각할 수 없어짐

## Perplexity (언어모델의 대표적 평가 지표)
1. 검증 손실을 조금 더해석하기 쉽게 변한한 것
  - 평균 토큰 당 손실을 구하고 거기다 지수를 취함
    - 왜 지수를 취하는가? 로그는 사람이 직관적 이해 어려움
 2. 모델이 다음 단어를 예측할 때 몇개의 단어 사이에서 망설이는지를 나타냄 
    - Perplexity = 1 이면 완벽한 모델이다. 항상 정확한 단어를 예측할 수있다
    - Perplexity = 어휘 크기라면 완전 무작위 상태, 아무것도 모르는 상태

    ### 요즘 Perplexity를 잘 사용하지 않는 이유
    1. 토크나이저에 따라 값이 달라짐
    2. 어휘 토큰의 개수에따라 상한 자체가 달라져 단순비교가 불가능
    3. 현재는 MMLU같은 벤치마크를 개발단계에서 사용

## MMLU 벤치마크
- 가장 유명한 벤치마크 (대학의학, 물리학, 천문학, 다양한 분야의 4지선다 문제로 구성)
    ### 평가방법 
    1. 네가지 선택지 각각이 나올 확률을 보고 높은것을 선택
    2. 모델에게 A, B, C, D 중 하나를 고르라고 하고 다음 토큰이 뭔지 봄

    - 평가의 함정이 존재함
 같은 MMLU도 평가 방법에 따라 결과가 크게달라짐

    ### 벤치마크 오염문제
    - 모델이벤치마크 테스트셋으로 학습했을 때 우리는 무엇으로 학습했는지 알 수 없다. 
    - 탐지방법: 대부분의 데이터셋은 무작위 순서가 아님 모델이 순서대로 생성할 때 역순으로 생성할 때보다 학률이 높으면 그 데이터가 학습셋에 있었을 가능성이 높다.
