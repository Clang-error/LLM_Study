# 언어 모델들은 왜 환각[hallucination](hallucination.md)을 일으키는가?
---
#### 요약목적: 대규모언어모델들의 환각이 왜 발생하는지에 대해서 간단히 알아보기 위함.

언어 모델의 환각은 이진 분류의 오류에서 기원함.
환각이라 표현하지만 인간지각 경험과는 다르다.

## 벤치마크에 대한 모델 최적화로 인한 환각조장
사람도 종종 그럴듯해 보이는 정보를 조작함.
불확실 할 때 객관식 시험에서 추측하거나, 서술형 시험에서 자신감이 낮음에도 불구하고 그럴듯한 답안을 제출하는 것처럼 언어모델도 이와 유사한 테스트를 통해 평가된다. 두 문제형식 경우 불확실할 때 추측하는 것이 정답에는 1점을 부여하고 공백이나 모름에는 0점을 부여하는 이진 점수 체계로 기대 점수를 최적화하는데, bluff는 종종 과도한 자신감과 구체성을 가지게된다. 많은 [LLM](LLM.md) 벤치마크는 정확도나 통과율과 같은 이진 지표를 사용하여 표준화된 인간 시험을 반영함. (보상관련모델정보 [[CS229_Summary#[reward-model](reward-model.md)|바로가기]]) 이러한 벤치마크에 대해 모델을 최적화하는 것은 환각을 조장하는 것이다.
언어모델의 대부분은 불확실성을 패널티로 부과하는 시험으로 평가된다.
따라서 모델들은 시험에서의 최적의 성능을내기위해 기울어져있다. 즉
대부분의 평가가 [[Alignment|정렬]]되지않았다는 뜻이다.

### 이진 채점 방식의 문제점
이진 채점이 환각을 측정하지 못한다
```
두 모델이 존재한다.
1번 모델은 99퍼의 완벽한 논리를 펼치다가 마지막 계산 실수를 낸 모델은 0점
2번 모델은 왜 그런지 전혀 모르겠고 그냥 확률상 가장 그럴듯한 단어를 찍었는데 맞췄으니 1점.
```
이진 채첨 방식의 가장 큰 문제이다. 중간 과정이나 얼마나 그럴듯하게 틀렸는지를 전혀 고려하지않고 오직 최종결과로 0 아니면 1로만 귀결해버린다. 또 다른 예시로
``` 
모델 A와 B가 존재한다. 
모델A는 불확실성을 올바르게 신호하고 결코 환각하지 않는 정렬된 모델이다 
모델B는 모델A와 유사하지만 불확실성을 나타내지 않고 불확실하다면 항상 추측을 하는 모델이다

모델B는 현재 벤치마크기반인 이진 채첨하에 A보다 우수할 것이다. 이는 불확실할때 추측하여 아무말이나 뱉는 모델B가 불확실성에 기피하지않았기에 더 우수한 점수를 받게된다. 이런 평가는 잘못되었다. 모델의 평가는 불확실할 때 기피에 대한 패널티를 부과하지 않도록 조정되어야 한다.
```
데이터에 모른다라는 답변을 아무리 학습시켜도 소용이없다. 시험 점수를 잘 받으려고 그럴듯하게 찍어 맞추는 행위를 보상하는 현 벤치마크 방식이 유지되는 한 AI의 환각이 절대 없어질 수 없다.
